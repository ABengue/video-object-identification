# VisionTrack: Video Object Identification & Interaction Analyzer

An asynchronous computer vision service that analyzes video files to identify objects, classify their motion (stationary vs. moving), detect human hand interactions, and extract transition keyframes.

## ⚡ Quick Start: Running Locally

### Prerequisites
- Python 3.9+
- macOS or Linux

### Launch Command
Run the following from the project root. It will create a virtual environment, install dependencies, pre-download YOLO model weights, and start the server automatically:

```bash
./run.sh
```

Once running, open your browser and navigate to:
👉 **[http://127.0.0.1:8000/](http://127.0.0.1:8000/)**

## 🎬 How to Use It

### Option A — Web Dashboard
1. Open **[http://127.0.0.1:8000/](http://127.0.0.1:8000/)** in your browser.
2. Click **"Analyze New Video"** in the sidebar, or drag and drop a video file onto the upload zone.
3. The task will appear in the **Analysis Queue** sidebar with a spinning progress indicator.
4. When processing completes (~30–60 seconds for a short video), click the task to view:
   - **Objects Catalogue** — every detected object with its motion state
   - **Interactive Timeline** — a Gantt-style chart of stationary/moving/interaction intervals
   - **Keyframe Gallery** — extracted JPEG frames at key transition moments
   - **JSON Payload** — the raw API output conforming to the schema below

### Option B — REST API (curl / Swagger)
The interactive Swagger UI is auto-generated by FastAPI and available at:
👉 **[http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)**

Upload a video and poll for results entirely via HTTP — no browser required.

### Seed Data
A sample video (`Sample Installation Video.mp4`) is included in the project root for immediate testing.

## 📡 API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/tasks` | Upload a video file. Returns a task object with a unique `id`. Accepts `multipart/form-data` with a `file` field. |
| `GET` | `/api/tasks` | List all tasks and their current status (`PENDING`, `PROCESSING`, `SUCCESS`, `FAILED`). |
| `GET` | `/api/tasks/{task_id}` | Get full task status and, when complete, the full `videoMetadata`, `objectsDetected`, and `keyFrames` payload. |

**Quick example with curl:**
```bash
# Upload a video
curl -X POST http://127.0.0.1:8000/api/tasks \
  -F "file=@Sample Installation Video.mp4"

# Poll status (replace with actual task id from above)
curl http://127.0.0.1:8000/api/tasks/<task_id>
```

## 📐 Core Algorithms & Mathematical Formulations

### 1. Motion Classification Logic
For a tracked object $i$, the center coordinate at frame $t$ is defined as $C_{i,t} = (x_t, y_t)$.
1. **Frame Displacement**:
   $$v_{i,t} = \sqrt{(x_t - x_{t-1})^2 + (y_t - y_{t-1})^2}$$
2. **Temporal Velocity Smoothing**: A moving average is calculated over a sliding window $W$ (default: $5$ frames) to suppress minor box fluctuations:
   $$\bar{v}_{i,t} = \frac{1}{W} \sum_{k=0}^{W-1} v_{i,t-k}$$
3. **Thresholding**: Frame states are classified as active (`1` = moving) if $\bar{v}_{i,t} \ge T_{\text{motion}}$ (default: $6.0$ pixels per frame). Otherwise, they are classified as (`0` = stationary).
4. **Majority Filter Smoothing**: We apply a temporal majority filter to sequences of binary states to eliminate rapid switching and unify intervals.

### 2. Hand-Object Proximity Interaction Logic
Instead of coarse bounding box overlaps, hands are tracked directly using 2D pose estimation joints:
- Left Wrist ($W_{\text{left}}$): Keypoint index $9$
- Right Wrist ($W_{\text{right}}$): Keypoint index $10$

For each person $P$ and object $i$ in frame $t$:
1. **Distance to Box Boundary**: The minimum Euclidean distance $d(W, \text{Box})$ from wrist coordinates $(x_w, y_w)$ with confidence $\ge 0.5$ to the nearest edge of the object's box $[x_{\min}, y_{\min}, x_{\max}, y_{\max}]$ is:
   $$d_x = \max(x_{\min} - x_w,\, 0,\, x_w - x_{\max})$$
   $$d_y = \max(y_{\min} - y_w,\, 0,\, y_w - y_{\max})$$
   $$d(W, \text{Box}) = \sqrt{d_x^2 + d_y^2}$$
2. **Proximity Flagging**: An active interaction is registered in frame $t$ if $\min(d(W_{\text{left}}, \text{Box}), d(W_{\text{right}}, \text{Box})) \le T_{\text{interaction}}$ (default: $70.0$ pixels).
3. **Occlusion Gap-Bridging**: Continuous interaction intervals allow a tolerance window of $10$ frames to bridge brief tracking dropouts.

### 3. Transition Keyframe Extraction
Keyframes are automatically written as `.jpg` images under two conditions:
- **State Transition**: The exact frame where an object transitions from `stationary` -> `moving`.
- **Peak Interaction Confidence**: The frame within an interaction interval where the wrist-to-box distance is minimized.

## 💡 Assumptions & Tradeoffs

1. **Wrist Estimation for Hands**: Wrist coordinates (left/right wrists, keypoints 9 and 10 from the COCO pose skeleton) are used as precise mathematical proxy points for hands, avoiding hand occlusion tracking issues.
2. **Database Concurrency**: Background workers instantiate and close isolated thread-safe database sessions, ensuring smooth parallel task execution without transaction deadlocks.
3. **Open-Vocabulary Detection**: YOLO-World v2 is used with custom class prompts (`spectrophotometer`, `cable`, `port`) rather than a fixed COCO class set, allowing the model to generalize to domain-specific objects out of the box without retraining.
